---------------------------------------------
# Oracle Select AI Agent 기본 개념
---------------------------------------------

### 오라클 Select AI Agent란

**Select AI Agent** 는 오라클 AI 데이터베이스 안에서 동작하는 AI 에이전트 프레임워크로써 이를 이용하여 대화형 또는 자율적으로 작업을 수행하는 AI 에이전트를 생성하고 관리할 수 있다. 에이전트의 정의, 관리, 그리고 도구(tool)는 PL/SQL 기반 API를 통해 구현된다.

#### Select AI Agent의 역할

Select AI Agent는 사용자의 요청을 받으면 다음과 같은 과정을 역할을 수행하게 된다.

1. 요청을 이해하고 분석(Reasoning)
2. 필요한 도구를 호출(Action)
3. 결과를 확인하고 다시 판단
4. 대화의 문맥과 이전 정보를 기억

이 과정은 ReAct(Reasoning + Acting) 방식의 Agent 구조를 사용하며, AI Profile에 지정된 **LLM(대형 언어모델)** 을 기반으로 동작한다.

**ReAct 패턴(반응 에이전트 패턴)**

Select AI Agent는 추론과 행동을 반복하는 루프를 가진 ReAct(추론 및 행동) 에이전트 패턴을 사용한다. 

Select AI Agent는 ReAct 패턴을 기반으로 생각하고, 도구를 선택하고, 결과를 관찰하고, 확신 있는 답변을 제시할 수 있을 때까지 과정을 반복한다. 사용자가 지정한 AI 프로필에 따라 LLM은 도구를 통해 추론과 행동을 번갈아 수행한다. 데이터베이스는 이러한 행동을 처리하고 관찰 결과를 반환한다.

다음은 각 반복 단계의 패턴이다.

|단계 |실행내용|
| :---| :--- |
|1. 질(Query) | 사용자가 질문을 하거나 요청을 한다. 에이전트는 이를 읽고 핵심 정보를 추출한 후 다음 단계를 계획할 준비를 한다.|
|2. 사고와 행동(Thought and Action) |에이전트는 여러 선택지를 추론하고, 도구를 선택한 후, 필요에 따라 데이터를 수집하거나 상태를 변경하기 위해 도구를 실행한다.|
|3. 관찰(Observation) |관찰에는 도구 또는 쿼리 결과, 확인 메시지 및 오류가 포함된다. 이러한 관찰 내용은 에이전트의 다음 추론 단계에 대한 입력으로 사용된다. 에이전트는 관찰 내용을 기록하고 결과가 다음 단계 또는 최종 응답을 뒷받침하는지 확인한다.|
|4. 최종 응답(Final Response) | 충분한 사고-행동 및 관찰 과정을 거친 후, 에이전트는 명확한 답변을 구성하고, 중요한 결정 사항을 설명하며, 다음 단계 또는 후속 조치를 공유한다.|

### Select AI Agent의 주요 특징

Select AI Agent는 데이터베이스 내부에서 AI 에이전트를 쉽게 만들고 운영할 수 있도록 하는 기능을 제공하며, 주요 특징은 다음과 같다.

- **통합된 지능 (Integrated Intelligence)**

AI 에이전트가 스스로 판단하고 작업을 수행하는 구조를 제공한다. 에이전트는 다음 과정을 반복하면서 문제를 해결하게 되고, 답변의 정확도를 높이고 불필요한 작업을 줄이며 대화를 자연스럽게 유지한다.

| ▼ |
|결과가 부족하면 계획 수정 (Reflection)|

- **유연한 도구 활용 (Flexible Tooling)**

Select AI Agent는 다양한 도구를 사용할 수 있다. 이 모든 기능을 별도의 AI 플랫폼이나 오케스트레이션 시스템 없이 데이터베이스 내부에서 바로 사용할 수 있다. 즉, 핵심 로직은 DB 안에 두면서 필요할 때 외부 시스템과 연동할 수 있습니다.

- **문맥을 이해하는 대화 (Context-Aware Conversations)**

단기 메모리와 장기 메모리를 유지하여 여러 번의 대화에서도 문맥을 유지하고, 응답을 개인화하며, 사용자 선호를 저장하고, 여러 차례의 대화 세션 동안 수정 및 확인을 위한 인간 참여(Human-in-the-loop) 제어를 지원한다. 

단기 메모리는 현재 대화의 일관성을 유지하고, 장기 메모리는 사용자 선호와 이전 결과를 기록하여 후속 상호작용과 인간 검토자의 감독을 지원한다. 이를 통해 여러 번의 대화를 이어가면서도 이전 내용과 문맥을 유지할 수 있다.

**Select AI Agent가 사용하는 메모리**

|메모리 영역 | 역할 |
| :--- | :---|
|Short-term memory | 현재 대화 문맥 유지 |
|Long-term memory | 이전 대화나 정보 기억 |

- **확장성과 보안 (Scalable & Secure)**

오라클 AI Database 내부에서 실행되기 때문에 데이터베이스가 제공하는 접근제어, 감사기능, 성능 특성을 그대로 활용한다. 따라서 데이터 이동을 줄이고 기업 환경 및 규제가 있는 환경에서 대규모로 표준화된 거버넌스를 적용할 수 있다. 

- **빠른 개발 (Faster Development)**

AI 에이전트를 만들고 운영할 때 SQL, PL/SQL과 같은 기존 데티이터베이스 기술을 그대로 사용할 수 있다. 특히 기존에 만들어둔 저장 프로시저, 데이터 로직을 그대로 활용할 수 있기 때문에 새로운 AI 인프라를 따로 구축하지 않고 빠르게 개발할 수 있다.

**Select AI Agnet의 계층 구조**

### Select AI Agent의 핵심 객체

**Agent(에이전트)**

**에이전트(Agent)** 는 특정 목적을 수행하기 위해 작업(Task)을 수행하는 실행 단위이다. 에이전트는 요청을 분석하고, 적절한 도구(Tool)를 선택하여 작업을 수행하며, 결과를 평가한 뒤 데이터베이스 컨텍스트(Context)을 기반으로 최종 응답을 생성한다. 

즉, 에이전트는 다음 기능을 수행하는 **구성된 작업자(worker)** 이다.

- 요청 분석
- 도구 선택
- 단계별 작업 실행
- 결과 평가
- 응답 생성

**Agent Team(에이전트 팀)**

**Agent Team(에이전트 팀)** 은 정의된 목적을 위해 작업을 수행하는 주체이며 에이전트 워크플로를 수행하는 하나 이상의 에이전트로 구성된다. 팀은 책임을 조율하고, 컨텍스트를 공유하며, 에이전트-작업 쌍을 실행하여 여러 단계로 이루어진 상호 작용을 안정적으로 완료한다. 즉, 복잡한 다단계 작업을 수행하기 위해 여러 에이전트가 협력하는 구조를 가진다.

Agent Team의 역할

- 에이전트 간 역할 분담
- 작업 흐름 조정
- 대화 및 작업 문맥 공유
- 여러 단계의 작업을 안정적으로 수행

**MCP 서버**

**MCP 서버(모델 컨텍스트 프로토콜 서버)** 는 애플리케이션이 생성형 AI 모델 및 도구와 상호 작용할 수 있도록하는 표준 인터페이스 서버이다. 이를 통해 일관된 프로토콜을 사용하여 AI 모델과 도구를 통합할 수 있다.

MCP 서버의 역할

- AI 모델 기능 제공
- 컨텍스트 정보 조회
- 확장 기능 연결

**Observation(관찰 결과)**

**Observation(관찰 결과)** 은 도구 실행에서 반환된 데이터(행, 메시지 또는 오류)를 말하며, 에이전트는 이를 기록하고 숙고 과정에서 평가하게 된다. Observation은 다음 추론 단계를 안내하고 최종 응답을 생성할 때 근거자료로 사용된다.

**Task(작업)**

**Task(작업)** 은 에이전트가 수행해야 하는 작업 단위를 말한다. 작업은 도구 선택(어떤 도구를 사용할지), 매개변수 매핑(어떤 파라미터를 전달할지) 및 실행 정책을 안내하며, 하위 단계에서 읽고 요약할 수 있는 결과를 생성한다.

**Tool(도구)**

**Tool(도구)** 는 특정 작업을 수행하는 기능이다. 예를 들어 데이터 업데이트, 문서 검색 또는 외부 서비스 호출과 같은 작업을 수행하는 기능이라고 말할 수 있다. 도구는 매개변수를 입력받고, 결정론적으로 또는 비결정론적으로 실행될 수 있으며, 추론을 위한 Observation(관찰 결과)를 반환한다.

**Agent Action(에이전트 동작)**

**액션(Action)** 은 도구(tool)를 특정 파라미터와 함께 실행하도록 하는 명령을 의미한다. 플랫폼 PLSQL 패키지를 이용하여 해당 **도구(Task)** 를 실행하고, **오류를 처리** 하며, 다음 추론 단계 반복(loop)을 위해 **관찰 결과(observation)** 를 반환한다. 

### Select AI Agent 활용 사례/시나리오

Select AI Agent는 다양한 역할과 업무 환경에서 실제 업무 자동화와 데이터 활용을 지원하는 시나리오를 제공한다.

주요 활용 방식은 다음과 같습니다.

- 외부 서비스와 연동
- PL/SQL 기반 업무 자동화
- 자연어 기반 데이터 조회

**1.외부 API 연동을 통한 응답 강화**

배송 상태 조회, 지식 문서, 메시징 서비스 등 REST 기반 외부 서비스를 연결할 수 있다. 에이전트는 1. 외부 서비스에서 데이터를 조회하고, 2. 데이터베이스의 내부 정보와 결합한 뒤, 3. 사용자에게 실행 가능한 안내 또는 결과를 제공한다. 즉 , 외부 데이터+내부 데이터를 결합하여 더 풍부한 응답을 생성한다.

**2.PLSQL 도구를 활용한 업무 자동화 에이전트 구축**

테이블 업데이트, 데이터 검증, 데이터 변환 등의 작업을 위하여 PL/SQL 도구를 활용하여 특정 업무에 특화된 에이전트를 만들 수 있다. 이를 통해 기존 PL/SQL 로직을 재사용 할수 있음은 물롤 로직을 데이터 가까이에서 실행하여 성능을 향상시키고 유지보수를 단순화할 수 있다.

이 경우 에이전트는 다음 역할을 수행한다.

- 작업 흐름 관리
- 파라미터 검증
- 실행 결과 기록
- 트랜잭션 제어 유지

**3.기업 데이터에 대한 대화형 접근 제공(Conversational Data Access)**

자연어 질문을 통해 데이터베이스 객체 기반의 기업 데이터를 조회할 수 있다. 

사용자는 자연어로 질문하면 
1. NL2SQL이 질문을 SQL로 변환하고, 
2. 데이터베이스에서 데이터를 조회한 후, 
3. 결과를 이해하기 쉬운 형태로 제공하게 된다. 

이 과정에서 데이터 접근 정책, 보안 규칙, 거버넌스 정책 관련한 요소들을 자연스럽게 적용 할 수 있다. 에이전트는 조회 결과를 요약하고 다음 단계 안내를 제공하면서 객체 접근 목록과 보안 규칙을 준수하게 된다.
